📋 CHECKLIST SETUP
------------------

✅ PASSO 1: SUPABASE (2 minuti)
   1. Vai su https://supabase.com e crea un account
   2. Crea un nuovo progetto (scegli nome, password, regione)
   3. Vai in "SQL Editor" → clicca "New query"
   4. Copia TUTTO il contenuto di "supabase-setup.sql"
   5. Incolla nell'editor e clicca "Run"
   6. Verifica che appaia "Success" (crea tabelle + dati demo)
   
   Salva queste credenziali:
   - URL: Settings → API → Project URL
   - Key: Settings → API → anon public (copia la chiave lunga)

✅ PASSO 2: NETLIFY (1 minuto)
   OPZIONE A - Drag & Drop (più veloce):
   1. Vai su https://www.netlify.com
   2. Fai login/signup
   3. Trascina la CARTELLA "budget-provvigioni-app" nell'area Deploy
   4. Attendi 30 secondi → fatto!
   
   OPZIONE B - Con Git:
   1. Crea repository GitHub
   2. Push del codice
   3. Connetti a Netlify
   4. Deploy automatico

✅ PASSO 3: CONFIGURAZIONE APP (30 secondi)
   1. Apri l'URL dell'app (fornito da Netlify)
   2. Si apre il modale di configurazione
   3. Incolla URL e Key di Supabase
   4. Clicca "Salva Configurazione"
   5. L'app si apre automaticamente con i dati!

🎉 HAI FINITO! L'app è pronta con dati demo già caricati.

🆘 PROBLEMI COMUNI
-------------------
❌ "Errore configurazione Supabase"
   → Controlla di aver copiato URL e Key corrette
   → Verifica che il progetto Supabase sia attivo

❌ "Nessun dato nella dashboard"
   → Vai in Supabase → SQL Editor
   → Esegui: SELECT * FROM agents; (deve tornare 8 righe)
   → Se vuoto, ri-esegui supabase-setup.sql

❌ "Import fallisce"
   → File CSV deve essere UTF-8
   → Prima riga = header (ym,agent_code,product_code,qty,amount)
   → Usa virgola come separatore (non punto e virgola)


📊 FORMATO CSV CORRETTO
------------------------
Budget:
ym,agent_code,product_code,qty,amount
2025-11,AGT-001,PRD-A,130,52000

Sales:
ym,agent_code,product_code,qty,amount,source,external_ref
2025-11,AGT-001,PRD-A,115,46000,ERP,FA-001234

-----------

1. Apri Console Browser (F12) → guarda errori in rosso
2. Vai in Supabase → Logs → SQL Queries → cerca errori
3. Verifica che le RLS (Row Level Security) siano attive
